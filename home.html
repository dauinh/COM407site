Evolving a fuzzy Xpilot controller

COM407 final project

View code

Project description

We want to create a fuzzy system to control Xpilot agent, based on the homework earlier this semester. There are two drawbacks in the previous system that we want to tackle this time:
(1) the values ranges in the old fuzzy system need to be carefully researched and calculated;
(2) the defuzzification process may require some rewriting.
In addition, as our old system focuses primarily on avoiding walls, we now want to add a separate set of fuzzy rules that control aiming and shooting of the agent. In addition, during the semester we’re quite intrigued by the concept of genetic algorithms. Therefore, if time permits, we may explore the possibility of combining GA and fuzzy systems, where we use GA to learn the correct value ranges for the fuzzy system.

How we did it

For the first phase, we created a fuzzy system that evaluate the risks of three behavior: wall avoidance, bullet dodging and shooting at enemy. Whichever with the highest risk get the execution priority. After a behavior has been chosen, it will go into the corresponding production system. We spent most of the time rewriting the system for bullet dodging.
In the second phase, we use a GA to train the fuzzy system of evaluating risk, using the chromosome size of 100 and the fitness function is the score of the agent. We did several training session; in the final training, we train 100 generations with each generation of 20 agents.

Result

We tried (ノ_<。)
The agent's only show a slight increase in competency after the training, and we figure out some possible sources of the problem:
(1) the production system is not very robust
(2) the fitness function needs modification (or we should have changed it to something else)
(3) the selection and crossover process might be buggy (cuz the agent seems to learn from the bad guys instead (╯°益°)╯彡┻━┻)
